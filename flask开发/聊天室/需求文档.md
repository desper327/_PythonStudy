# Flask-PySide6 聊天室系统需求文档

## 1. 项目概述

### 1.1 项目名称
Flask-PySide6 实时聊天室系统

### 1.2 项目目标
开发一个基于Flask后端和PySide6桌面客户端的聊天室应用，支持用户私聊和群组聊天，具备完整的用户管理和群组管理功能。

### 1.3 技术栈
- **后端框架**: Flask + Flask-SocketIO (实时通信)
- **数据库**: MySQL
- **前端框架**: PySide6 (Qt for Python)
- **通信协议**: WebSocket (Socket.IO)
- **ORM**: SQLAlchemy
- **认证**: JWT (JSON Web Token)

## 3. 功能需求

### 3.1 用户管理模块

#### 3.1.1 用户注册
- **输入**: 用户名、密码、邮箱（可选）、昵称
- **验证**: 
  - 用户名唯一性检查
  - 密码强度验证（至少6位）
  - 邮箱格式验证
- **输出**: 注册成功/失败信息

#### 3.1.2 用户登录
- **输入**: 用户名、密码
- **验证**: 
  - 用户名和密码匹配
  - 用户状态是否正常
- **输出**: JWT Token、用户信息

#### 3.1.3 用户信息管理
- **查看个人信息**: 用户名、昵称、邮箱、注册时间
- **修改个人信息**: 昵称、邮箱、头像
- **修改密码**: 旧密码验证 + 新密码设置
- **注销账号**: 删除用户数据（需确认）

#### 3.1.4 用户列表
- **搜索用户**: 按用户名或昵称搜索
- **查看用户信息**: 查看其他用户的公开信息
- **添加好友**: 发送好友请求（可选功能）

### 3.2 群组管理模块

#### 3.2.1 创建群组
- **输入**: 群组名称、群组描述、群组类型（公开/私有）
- **验证**: 
  - 群组名称唯一性
  - 群组名称长度限制（2-30字符）
- **输出**: 群组创建成功，创建者自动成为群主

#### 3.2.2 群组列表
- **我的群组**: 显示已加入的所有群组
- **公开群组**: 显示所有公开群组
- **搜索群组**: 按群组名称搜索

#### 3.2.3 加入/退出群组
- **加入群组**: 
  - 公开群组：直接加入
  - 私有群组：需要群主或管理员审批
- **退出群组**: 主动退出（群主需转让群主权限）

#### 3.2.4 群组成员管理
- **查看成员列表**: 显示所有群组成员
- **踢出成员**: 仅群主和管理员可操作
- **设置管理员**: 仅群主可操作
- **转让群主**: 仅群主可操作

#### 3.2.5 群组信息管理
- **修改群组信息**: 群组名称、描述（仅群主和管理员）
- **解散群组**: 仅群主可操作

### 3.3 消息管理模块

#### 3.3.1 私聊消息
- **发送消息**: 
  - 文本消息
  - 表情（可选）
  - 文件（可选，后期功能）
- **接收消息**: 实时接收对方发送的消息
- **消息状态**: 
  - 发送中
  - 已发送
  - 已读（可选）
  - 发送失败

#### 3.3.2 群组消息
- **发送群组消息**: 
  - 文本消息
  - @提醒功能
  - 群公告（管理员功能）
- **接收群组消息**: 实时接收群组内所有消息
- **消息过滤**: 
  - 屏蔽某些成员（可选）

#### 3.3.3 消息历史
- **查看历史消息**: 
  - 分页加载
  - 按时间排序
  - 本地缓存（提高加载速度）
- **搜索消息**: 按关键词搜索
- **删除消息**: 
  - 删除单条消息（仅本地）
  - 清空聊天记录

#### 3.3.4 消息通知
- **桌面通知**: 新消息弹窗提示
- **未读消息**: 
  - 未读消息计数
  - 未读消息标记
- **消息提示音**: 可开启/关闭

### 3.4 系统设置模块

#### 3.4.1 连接设置
- **服务器地址**: 配置后端服务器地址
- **端口设置**: 配置服务器端口
- **连接状态**: 显示在线/离线状态

#### 3.4.2 通知设置
- **消息通知**: 开启/关闭
- **提示音**: 开启/关闭
- **桌面弹窗**: 开启/关闭

#### 3.4.3 界面设置
- **主题切换**: 亮色/暗色主题
- **字体大小**: 调整界面字体大小
- **语言**: 中文/英文（可选）

## 4. 数据库设计

### 4.1 用户表 (users)
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | INT | 用户ID | PRIMARY KEY, AUTO_INCREMENT |
| username | VARCHAR(50) | 用户名 | UNIQUE, NOT NULL |
| password_hash | VARCHAR(255) | 密码哈希 | NOT NULL |
| nickname | VARCHAR(50) | 昵称 | NOT NULL |
| email | VARCHAR(100) | 邮箱 | UNIQUE, NULL |
| avatar | VARCHAR(255) | 头像URL | NULL |
| status | ENUM | 用户状态 | 'active', 'banned', 'deleted' |
| created_at | DATETIME | 创建时间 | DEFAULT CURRENT_TIMESTAMP |
| updated_at | DATETIME | 更新时间 | ON UPDATE CURRENT_TIMESTAMP |

### 4.2 群组表 (groups)
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | INT | 群组ID | PRIMARY KEY, AUTO_INCREMENT |
| name | VARCHAR(50) | 群组名称 | UNIQUE, NOT NULL |
| description | TEXT | 群组描述 | NULL |
| type | ENUM | 群组类型 | 'public', 'private' |
| owner_id | INT | 群主ID | FOREIGN KEY(users.id) |
| created_at | DATETIME | 创建时间 | DEFAULT CURRENT_TIMESTAMP |
| updated_at | DATETIME | 更新时间 | ON UPDATE CURRENT_TIMESTAMP |

### 4.3 群组成员表 (group_members)
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | INT | 成员ID | PRIMARY KEY, AUTO_INCREMENT |
| group_id | INT | 群组ID | FOREIGN KEY(groups.id) |
| user_id | INT | 用户ID | FOREIGN KEY(users.id) |
| role | ENUM | 成员角色 | 'owner', 'admin', 'member' |
| joined_at | DATETIME | 加入时间 | DEFAULT CURRENT_TIMESTAMP |
| UNIQUE | (group_id, user_id) | 联合唯一索引 |  |

### 4.4 消息表 (messages)
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | INT | 消息ID | PRIMARY KEY, AUTO_INCREMENT |
| sender_id | INT | 发送者ID | FOREIGN KEY(users.id) |
| receiver_id | INT | 接收者ID | FOREIGN KEY(users.id), NULL |
| group_id | INT | 群组ID | FOREIGN KEY(groups.id), NULL |
| message_type | ENUM | 消息类型 | 'text', 'image', 'file' |
| content | TEXT | 消息内容 | NOT NULL |
| is_read | BOOLEAN | 是否已读 | DEFAULT FALSE |
| created_at | DATETIME | 发送时间 | DEFAULT CURRENT_TIMESTAMP |
| INDEX | (sender_id) | 发送者索引 |  |
| INDEX | (receiver_id) | 接收者索引 |  |
| INDEX | (group_id) | 群组索引 |  |

**注意**: receiver_id 和 group_id 二选一，私聊消息填 receiver_id，群组消息填 group_id

## 5. API接口设计

### 5.1 认证接口

#### POST /api/auth/register - 用户注册
**请求体**:
```json
{
  "username": "string",
  "password": "string",
  "nickname": "string",
  "email": "string (optional)"
}
```
**响应**:
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "user_id": 1,
    "username": "string"
  }
}
```

#### POST /api/auth/login - 用户登录
**请求体**:
```json
{
  "username": "string",
  "password": "string"
}
```
**响应**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "jwt_token_string",
    "user": {
      "id": 1,
      "username": "string",
      "nickname": "string",
      "email": "string",
      "avatar": "string"
    }
  }
}
```

### 5.2 用户接口

#### GET /api/users/me - 获取当前用户信息
**Headers**: Authorization: Bearer {token}

#### PUT /api/users/me - 更新当前用户信息
**Headers**: Authorization: Bearer {token}
**请求体**:
```json
{
  "nickname": "string",
  "email": "string",
  "avatar": "string"
}
```

#### GET /api/users/search?keyword={keyword} - 搜索用户

#### GET /api/users/{user_id} - 获取指定用户信息

### 5.3 群组接口

#### POST /api/groups - 创建群组
**请求体**:
```json
{
  "name": "string",
  "description": "string",
  "type": "public/private"
}
```

#### GET /api/groups - 获取群组列表（我的群组）

#### GET /api/groups/public - 获取公开群组列表

#### GET /api/groups/{group_id} - 获取群组详情

#### POST /api/groups/{group_id}/join - 加入群组

#### DELETE /api/groups/{group_id}/leave - 退出群组

#### GET /api/groups/{group_id}/members - 获取群组成员列表

#### DELETE /api/groups/{group_id}/members/{user_id} - 踢出成员

### 5.4 消息接口

#### GET /api/messages/private/{user_id} - 获取与指定用户的私聊历史
**Query参数**: page, per_page

#### GET /api/messages/group/{group_id} - 获取群组聊天历史
**Query参数**: page, per_page

### 5.5 WebSocket事件

#### 客户端 -> 服务器

- **connect**: 连接建立
- **authenticate**: 认证 `{token: "jwt_token"}`
- **send_private_message**: 发送私聊消息 `{receiver_id: 1, content: "text"}`
- **send_group_message**: 发送群组消息 `{group_id: 1, content: "text"}`
- **disconnect**: 断开连接

#### 服务器 -> 客户端

- **authenticated**: 认证成功
- **new_private_message**: 新私聊消息 `{message_id, sender, content, created_at}`
- **new_group_message**: 新群组消息 `{message_id, group_id, sender, content, created_at}`
- **error**: 错误信息 `{message: "error description"}`
- **user_online**: 用户上线 `{user_id}`
- **user_offline**: 用户下线 `{user_id}`

## 6. 非功能需求

### 6.1 性能要求
- 消息发送延迟 < 500ms
- 支持至少100个并发用户
- 历史消息加载时间 < 2s
- 客户端启动时间 < 3s

### 6.2 安全要求
- 密码使用bcrypt加密存储
- JWT Token有效期24小时
- 防止SQL注入
- 防止XSS攻击
- 防止CSRF攻击

### 6.3 可靠性要求
- 消息持久化存储
- 断线自动重连
- 消息发送失败重试机制
- 数据库定期备份

### 6.4 可用性要求
- 界面简洁直观
- 操作响应及时
- 错误提示清晰
- 支持键盘快捷键

## 8. 技术难点和解决方案

### 8.1 实时通信
**难点**: 保证消息的实时性和可靠性
**解决方案**: 
- 使用Flask-SocketIO实现WebSocket
- 实现心跳机制检测连接状态
- 断线重连和消息重发机制

### 8.2 消息持久化
**难点**: 大量消息的存储和查询效率
**解决方案**:
- 数据库索引优化
- 消息分页加载
- 客户端本地缓存

### 8.3 并发处理
**难点**: 多用户同时发送消息
**解决方案**:
- 使用消息队列（可选）
- 数据库连接池
- 异步处理

### 8.4 跨线程UI更新
**难点**: PySide6 UI更新必须在主线程
**解决方案**:
- 使用Qt的信号槽机制
- QThread管理网络通信线程
